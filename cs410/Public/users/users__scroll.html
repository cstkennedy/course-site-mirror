User Roles and Stories

Review the related instructional material from the CS 350 content as needed.

You are tasked with the identification of User Roles for the Real World Product.
These should include Administrator, along with clearly identified additional level of user.

For each, start the process of creating their user stories. For example:

As an Administrator I need to be able to add a new student account.

For each User Story, designate it as either a need or a wish. A need indicates that this story must have an associated design and implementation. A wish indicates that the customer/mentor will be able to use the solution adequately without this capability/feature.
