1 Overview

All objectives are based on Bloom’s Taxonomy, https://ww1.odu.edu/content/dam/odu/col-dept/teaching-learning/docs/blooms-taxonomy-handout.pdf. Each group of objectives is split into Course Objectives and Honors Objectives where the

  1. former is required of all students
  2. latter is required of Honors College students

Note that all Honors Objectives require an understanding at least one level higher in Bloom’s Taxonomy than the corresponding Course Objective(s).

2 Listing of Objectives

This section lists selected Course Objectives and the corresponding Honors Objectives

Course Objectives

  1. (Understanding) Describe the use of an inner class in the context of abstraction and encapsulation.

Honors Objectives

  1. (Analyzing) Examine how an inner class can simplify an existing piece of code.
  2. (Analyzing) Examine how an inner class can potentially complicate an existing piece of code.

Course Objectives

  1. (Understanding) Explain and track data lifetimes.
  2. (Understanding) Explain ownership.

Honors Objectives

  1. (Analyzing) Compare (C++/Java) pass-by-value (pass-by-copy) to (Rust) pass-by-move
  2. (Analyzing) Compare (C++/Java) pass-by-reference to (Rust) pass-by-borrow

Course Objectives

  1. (Understanding) Explain what a pointer stores.
  2. (Understanding) Explain the difference between a pointer and a reference variable.
  3. (Analyzing) Analyze memory deallocation pitfalls.
  4. (Understanding) Explain and track data lifetimes.
  5. (Understanding) Explain why memory management can be a non-trivial problem.
  6. (Understanding) Summarize the motivation behind smart pointers.
  7. (Understanding) Explain ownership.
  8. (Understanding) Summarize garbage collection.

Honors Objectives

  1. (Evaluating) Evaluate the lifetime of one or more objects in an exiting codebase through use of a UML Sequence Diagram.

Course Objectives

  1. (Understanding) Discuss the Factory Model.
  2. (Understanding) Discuss the Builder Pattern.

Honors Objectives

  1. (Analyzing) Compare the Factory Pattern and Builder Pattern, specifically examine cases where one (or both) patterns will simplify the design of an existing codebase.

3 Evaluation

Student understanding will be evaluated through a combination of written examination (via Canvas) and a whiteboard/technical interview (via Zoom).
